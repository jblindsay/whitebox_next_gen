Orthorectification

DEM-based geometric correction of raw imagery using RPC camera model. Removes terrain relief displacement for georeferenced orthoimage output.

remote-sensing geometric-correction orthorectification rpc dem

Parameters

NameDescriptionRequiredDefault
input_rasterInput raw (uncorrected) image raster path. RPC metadata must be embedded in the file or present in an adjacent .RPB sidecar.Requiredraw_image.tif
input_demDEM raster used to obtain terrain elevation for each output pixel. Should cover the full scene extent. If not in geographic coordinates, the tool reprojects as needed.Requireddem.tif
output_epsgEPSG code of the output coordinate reference system (default 4326 geographic; specify a projected CRS like 32617 for UTM zone 17N).Optional4326
output_resolutionOutput pixel resolution in the output CRS units (e.g., degrees for geographic, metres for projected). Defaults to the approximate GSD of the input image.OptionalNone
resample_methodResampling method for output: nearest, bilinear (default), cubic.Optionalbilinear
nodata_valueNo-data value to assign to pixels with no valid source coverage (default same as input).OptionalNone
outputOutput orthoimage raster path.Optionalorthorectified.tif

Examples

Orthorectify a Sentinel-2 band using a co-regional DEM, outputting a UTM projected orthoimage.

wbe.orthorectification(input_dem='dem.tif', input_raster='raw_image.tif', nodata_value=None, output='orthorectified.tif', output_epsg=32617, output_resolution=10.0, resample_method='bilinear')
